# ThermoTwin

### A High-Fidelity Digital Twin for Kia Seltos Thermal Management

ThermoTwin is not just a dashboard; it is a high-fidelity Digital Twin designed to simulate, monitor, and optimize the thermal environment of a Kia Seltos cabin. By integrating real-time sensor telemetry with advanced signal processing and physics-based modeling, ThermoTwin provides a production-grade interface for visualizing complex cabin thermodynamics and predictive thermal control.

---

## 🚀 Key Features

### 📊 Multi-Zone Diagnostics
High-precision trajectory plotting of cabin temperatures across three logical zones. The interface features predictive "look-ahead" dashed lines generated by our machine learning pipeline to forecast thermal states 60-120 seconds in the future.

### 🗺️ High-Fidelity Thermal Mapping
A top-down vertical cabin visualization featuring a radial-masked car chassis blueprint. Dynamic "thermal blobs" pulse and shift color based on real-time temperature data, while green occupancy borders provide instant visual cues for passenger presence.

### 📡 Advanced Signal Processing (Kalman Filtering)
Integrated Recursive Least Squares and Kalman Filtering to denoise telemetry from simulated DHT11/LM35 sensor arrays. The system calculates innovation covariance and updates state vectors in real-time to ensure the Digital Twin remains synchronized with physical reality.

### 🧪 Physics-Engine Core
A custom-built RC (Resistor-Capacitor) Thermal Model that simulates heat transfer by accounting for:
*   **Solar Load ($Q_{solar}$):** Radiative heat transfer based on intensity and time of day.
*   **Occupancy ($Q_{occupancy}$):** Body heat generation per seat ($0.08°C/step$).
*   **HVAC Efficiency:** Dynamic AC power throttling based on setpoint deviation.

## 📐 Mathematical Architecture

### State Estimation (Kalman Filter)
The system employs a Kalman filter to estimate the true temperature ($x$) by weighing the physics-model prediction against noisy sensor measurements ($z$):

$$
K_k = \frac{P_{k|k-1} H^T}{H P_{k|k-1} H^T + R}
$$

$$
\hat{x}_k = \hat{x}_{k|k-1} + K_k(z_k - H \hat{x}_{k|k-1})
$$

### RC Thermal Model
The cabin is modeled as a thermodynamic circuit where temperature change is a function of thermal capacitance ($C$) and resistance ($R$):

$$
T_{next} = T_{curr} + \frac{dt}{C} \times \left( Q_{ac} + Q_{sun} + Q_{body} - \frac{T_{curr} - T_{amb}}{R} \right)
$$

## 🔌 API Endpoints

| Endpoint | Method | Description |
| :--- | :--- | :--- |
| `/api/state` | `GET` | Returns the current Digital Twin state (Zones, ML Predictions, System Status). |
| `/api/controls` | `POST` | Updates active system controls (Setpoint, AC Toggle, Solar Load, Occupancy). |
| `/api/ducts` | `GET` | Returns current HVAC duct opening percentages and flow strategy. |

## ⚙️ Installation & Setup

### 1. Backend Setup (Flask)
```bash
cd backend
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate
pip install -r requirements.txt
python app.py
```

### 2. Frontend Setup (React + Vite)
```bash
cd frontend
npm install
npm run dev
```

## 📡 Control Loop & Synchronization
The system operates on a 2-second polling window (`intervalMs = 2000`). The frontend `useTwinPolling` hook ensures that the UI state, technical charts, and thermal maps are perfectly synchronized with the Flask physics engine's execution steps.
